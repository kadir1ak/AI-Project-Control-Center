# Ideation

## Amaç

Ideation ekranı, kullanıcının ham proje fikrini yapılandırılmış ürün fikrine dönüştürür.

## Input

Kullanıcı serbest metin girer:

```text
Klinikler için randevu, hasta takip ve ödeme modülü olan bir SaaS yapmak istiyorum.
```

## Output

Sistem şu çıktıları üretir:

- Problem tanımı.
- Hedef kullanıcı.
- Kullanıcı personları.
- Ana özellikler.
- MVP kapsamı.
- MVP dışı kapsam.
- Gelir modeli varsayımı.
- Riskler.
- Varsayımlar.
- İlk ürün kapsamı.

## İş Akışı

1. Kullanıcı fikri yazar.
2. Sistem eksik bilgileri varsayımlarla tamamlar.
3. Kullanıcı üretilen özeti düzenler.
4. Onaylanan fikir PRD Editor'a kaynak olur.

## MVP Kuralları

- Ideation çıktısı doğrudan kod üretmemelidir.
- Çıktı, PRD ve backlog üretimi için kaynak doküman olmalıdır.
- Varsayımlar açıkça işaretlenmelidir.
- Kullanıcı düzenlemeden sonraki versiyon kayıt altına alınmalıdır.

## Kabul Kriterleri

- Kullanıcı tek metinle fikir girebilir.
- Sistem yapılandırılmış fikir özeti üretir.
- Sistem risk ve varsayımları ayrı gösterir.
- Kullanıcı çıktıyı PRD üretimi için onaylayabilir.
