# PRD Editor

## Amaç

PRD Editor, ürün gereksinim dokümanının üretildiği, düzenlendiği ve sonraki aşamalara kaynak yapıldığı alandır.

## PRD Bölümleri

- Ürün adı.
- Ürün amacı.
- Hedef kullanıcılar.
- Problem tanımı.
- Çözüm önerisi.
- MVP kapsamı.
- MVP dışı kapsam.
- Ana özellikler.
- Kullanıcı akışları.
- Başarı metrikleri.
- Teknik varsayımlar.
- Riskler.
- Acceptance criteria.

## Özellikler

- AI ile PRD üretme.
- Bölüm bazlı yeniden üretme.
- Manuel düzenleme.
- Versiyon geçmişi.
- PRD'den backlog üretme.
- PRD'den mimari özet üretme.

## MVP Davranışı

İlk sürümde PRD markdown olarak tutulabilir. Rich text editör şart değildir; temiz bir markdown editor yeterlidir.

## PRD'den Sonraki Çıktılar

PRD onaylanınca sistem şunları üretebilir:

- Epic listesi.
- User story listesi.
- Acceptance criteria.
- İlk teknik gereksinim seti.
- Risk ve açık soru listesi.

## Kabul Kriterleri

- PRD üretimi tek aksiyonla başlatılır.
- Kullanıcı PRD'yi düzenleyebilir.
- Sistem PRD versiyonunu saklar.
- Backlog üretimi sadece onaylanmış PRD üzerinden yapılır.
